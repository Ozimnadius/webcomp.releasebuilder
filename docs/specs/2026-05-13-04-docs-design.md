**Language:** Русский (весь PHPDoc и README).

## Задача 1: vendor.php + install/index.php + controller PHPDoc

### Удалить `include/vendor.php`

Файл больше не нужен — удалить через `git rm include/vendor.php`.

### `install/index.php`

Добавить класс-блок над `class webcomp_releasebuilder`:

```php
/**
 * Инсталлятор модуля webcomp.releasebuilder.
 *
 * Регистрирует модуль в Bitrix, копирует файлы интерфейса в публичные директории
 * и заполняет поля вендорной конфигурации для Bitrix Marketplace.
 */
```

Добавить PHPDoc к методам:

| Метод | Описание |
|---|---|
| `__construct()` | Инициализирует поля модуля из языкового файла и `install/version.php` |
| `DoInstall()` | Устанавливает модуль: копирует файлы и регистрирует в Bitrix |
| `DoUninstall()` | Деинсталлирует модуль: удаляет файлы и снимает регистрацию |
| `InstallFiles()` | Копирует admin/js/css из install/ в публичные директории Bitrix |
| `UnInstallFiles()` | Удаляет скопированные admin/js/css из публичных директорий |

### `lib/controller/releasebuilder.php`

Добавить `@param`/`@return` к пяти публичным action-методам:

**`versionAction(string $module): ?array`**
- `@param string $module` — идентификатор модуля (например, `'webcomp.market'`)
- `@return array{VERSION: string, VERSION_DATE: string}|null` — данные версии или null при ошибке

**`getConfigAction(string $module): ?array`**
- `@param string $module`
- `@return array{patterns: string[], templateName: string, availableTemplates: string[], suggestedTemplate: string, moduleType: string}|null`

**`saveConfigAction(string $module, string $patterns, string $templateName): ?bool`**
- `@param string $module`
- `@param string $patterns` — паттерны исключений, по одному на строку (разделитель `\n`)
- `@param string $templateName` — название шаблона; допустимые символы: `[a-zA-Z0-9_\-.]`
- `@return bool|null` — true при успехе, null при ошибке валидации

**`searchAction(...): ?array`**
- `@param string $module`
- `@param string $newVersion` — целевая версия в формате `major.minor.patch`
- `@param string $version` — текущая установленная версия
- `@param string $type` — тип модуля (`'regular'` или `'template'`)
- `@param string $date` — дата последнего обновления; файлы новее этой даты попадут в архив
- `@param string $templateName`
- `@return string[]|null` — список абсолютных путей изменённых файлов или null при ошибке

**`prepareArchiveAction(...): ?string`**
- `@param string $module`
- `@param string $newVersion`
- `@param string $version`
- `@param string $type`
- `@param string $date`
- `@param string $description` — текст описания обновления (кодируется в Windows-1251)
- `@param string $templateName`
- `@param string $selectedFiles` — JSON-массив абсолютных путей к выбранным файлам
- `@return string|null` — путь к архиву относительно docRoot или null при ошибке

Добавить PHPDoc к приватным методам:

**`moduleService(): ModuleService`**
```
Возвращает (или создаёт) единственный экземпляр ModuleService для текущего запроса.
@return ModuleService
```

**`filterConfig(): FilterConfig`**
```
Возвращает (или создаёт) единственный экземпляр FilterConfig для текущего запроса.
@return FilterConfig
```

Субагент читает все 13 файлов в `lib/` и проверяет:
- Соответствуют ли типы в `@param`/`@return`/`@throws` фактическим сигнатурам

Правки — минимальные, только расхождения. Один коммит.

## Задача 3: README.md

Создать `README.md` в корне модуля. За основу взять standalone-версию (`release-builder/README.md`), адаптировав под модуль:

### Разделы и изменения

**Быстрый старт**
- Точка входа: Bitrix-админка → раздел «Release Builder» (не прямой URL)
- Шаги те же, но без упоминания `http://market.web-komp.local/release-builder/`

**Настройка фильтров**
- Таблица форматов паттернов — без изменений
- Путь хранения: `upload/release-builder/config.json` (вместо `release-builder/config.json`)

**Шаблонные решения**
- Путь к шаблону `updater.php.tpl`: `bitrix/modules/webcomp.releasebuilder/install/updater.php.tpl`
- Таблица плейсхолдеров — без изменений

**Архитектура**
- HTTP-слой: Bitrix AJAX-контроллер (`lib/controller/releasebuilder.php`) вместо `ajax.php`
- Диаграмма обновляется: `HTTP (Bitrix AJAX)` → `Controller` → сервисы

**Структура файлов**
Полностью новая — Bitrix-модуль:

```

├── install/
│   ├── index.php          # Инсталлятор модуля
```
